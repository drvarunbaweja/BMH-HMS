# BMH HMS — Project Reference Document
## Baweja Multispeciality Hospital Management System
### Last Updated: March 2026 | Current Version: v49

---

## Project Overview

A **single-file HTML Hospital Management System** for Baweja Multispeciality Hospital (BMH),
operating across two branches: Chandigarh (CHD) and Ropar (RPR).

**Architecture:** Everything in one `.html` file — HTML, CSS, JavaScript, Firebase SDK,
base64 letterhead image (~70KB embedded). No build process, no server, no dependencies
to install. Open in any modern browser.

**Firebase:** Google Firebase Realtime Database (compat SDK v10.7.0)
- Project ID: `bmh-hms`
- Database URL: `https://bmh-hms-default-rtdb.asia-southeast1.firebasedatabase.app`
- Storage bucket: `bmh-hms.firebasestorage.app`

## Core Function Reference

### Patient Lifecycle

| Function | Signature | What It Does |
|----------|-----------|-------------|
| `registerPatient()` | `()` | Reads reception form, creates patient object, pushes to PATIENTS, saves to Firebase `patients/`, generates token number |
| `openPatient(bmhId)` | `(string)` | Sets `window.CURRENT_PATIENT`, clears ALL form fields/selects/textareas/checkboxes/chips across all dept pages, navigates to patient's dept, loads past visits |
| `markSeen(bmhId)` | `(string)` | Sets `p.seen=true, p.status='seen', p.dilated=false`, saves to Firebase, re-renders queue |
| `markDilated(bmhId, name)` | `(string, string)` | Sets `p.dilated=true`, saves timestamp + Firebase, re-renders queue |
| `saveVisit(dept)` | `('ophtho'\|'obg'\|'psych'\|'skin')` | Collects all dept-specific form data + RX_DRUGS snapshot, saves to `visits/{bmhId}/{key}` |
| `loadPastVisits(bmhId, dept)` | `(string, string)` | Fetches all visits for patient from Firebase, renders history cards |

### Queue & Navigation

| Function | What It Does |
|----------|-------------|
| `renderDocQueue()` | Filters PATIENTS by `CURRENT_USER.dept` (admin sees all). Populates Active / Dilated / Cross-Refer / IPD / Done tabs. Dilated tab hidden for non-ophtho |
| `buildQCard(p)` | Returns queue card HTML with inline icons: ✓ (all), 💧 (ophtho only), ↔️ (all), 🛏️ (all) |
| `nav(id, el)` | Navigates to page by ID, triggers page-specific render functions. Multiple nav overrides chained (`_navV38`, `_navV38b`) |
| `renderDashboard()` | Updates OPD count, collection, pending, appointments, low-stock count |

### Cross-Refer Workflow

| Function | What It Does |
|----------|-------------|
| `openXRefModal(bmhId)` | Populates patient selector, opens `#m-xref` modal |
| `doXRef()` | **Free:** changes `p.dept` immediately, patient appears in target queue. **Paid:** pushes to PAY_REQUESTS with `xref:true, xrefDept, xrefDoctor`; patient moves to queue only after `markPaid()` |
| `markPaid(id)` | If `pr.xref===true` → routes patient to `pr.xrefDept`, sets `p.status='waiting'`. Always saves to Firebase |

### IPD Workflow

| Function | What It Does |
|----------|-------------|
| `openIPDFromQueue(bmhId)` | Pre-selects patient in IPD modal, opens `#m-ipd-admit` |
| `confirmIPDAdmit()` | Creates IPD entry object, pushes to `window.IPD_PATIENTS`, saves to Firebase `ipdPatients/`, sets `p.ipdAdmitted=true`, logs audit |
| `renderIPD()` | Syncs `window.IPD_PATIENTS` → local array, renders IPD list page |

### OT Module

| Function | What It Does |
|----------|-------------|
| `renderOTList()` | Renders OT cases with `otCaseCard()`. Filters by date/surgeon selectors |
| `renderOTListSafe()` | Safe wrapper around `renderOTList()`, called by nav override |
| `openOTCase(id)` | Sets `activeOTCase`, prefills timer inputs from `c.timings`, prefills notes (no hardcoded text), renders detail panel with WHO status |
| `addOTCase()` | Creates case object, pushes to `OT_CASES`, saves to Firebase `otCases/` |
| `signOffWHO(phase)` | Validates all `.who-cb[data-checked="1"]` in tab, marks phase complete with timestamp + user, saves to Firebase |
| `toggleWHO(el)` | Toggles `data-checked` attribute on WHO checklist checkbox |
| `stampTime(id)` | Sets input to current time, saves to `activeOTCase.timings[key]`, syncs Firebase |
| `calcOTDurations()` | Calculates anaes→incision, procedure time, total OT time from time inputs |
| `saveOTNotes()` | Reads notes/complications/bloodLoss from form, saves to Firebase |
| `updateOTStatus(id, status)` | Updates case status + `lastUpdated`, saves to Firebase |

### Printing

### Login & User Management

### Firebase Helpers
```javascript
fbSet(path, data)       // FBDB.ref(path).set(data) — returns Promise
fbUpdate(path, data)    // FBDB.ref(path).update(data) — returns Promise
fbPush(path, data)      // FBDB.ref(path).push(data) — returns Promise (auto-key)
fbOnce(path, cb)        // FBDB.ref(path).once('value') → cb(val)
fbOn(path, cb)          // FBDB.ref(path).on('value') → real-time listener
```

All helpers silently resolve if `window.FBDB` is undefined (Firebase not loaded),
so offline use doesn't throw errors.

## Script Block Structure

The HTML file has **7 `<script>` blocks**:

| # | Size | Content |
|---|------|---------|
| 0 | ~70KB | Letterhead base64 image assignment + DOMContentLoaded show letterhead |
| 1 | empty | (Firebase CDN) |
| 2 | empty | (Firebase CDN) |
| 3 | ~405KB | **Main application** — all data, all functions, all logic |
| 4 | empty | |
| 5 | empty | |
| 6 | ~21KB | Firebase init + `initFirebase()` IIFE |

The main script (block 3) is a single `<script>` tag with no `type="module"` —
everything is global scope. Function declarations are hoisted, so order largely
doesn't matter, but several functions use `window.nav` overrides chained via
`_navV38`, `_navV38b` etc.

## Known Limitations & Future Work

### Security
- [ ] Passwords stored in plain text in Firebase `userSettings/` — should use
      Firebase Authentication or at minimum server-side hashing
- [ ] Firebase Realtime DB rules should restrict read/write to authenticated users
- [ ] No session timeout — browser refresh keeps user logged in via localStorage

### Features Incomplete / Partially Done
- [ ] **Taper medicines** — `addTaperRow()` exists in JS but taper UI on
      prescription is not fully implemented

## Bugs to Watch For in Future Development

1. **Nav override chain** — `window.nav` is overridden multiple times via
   `_navV38`, `_navV38b`. Adding more overrides must follow the same pattern
   to avoid breaking navigation

2. **`renderRxDrugs()` dept detection** — detects dept by
   `document.querySelector('.page.active')?.id`. If called before nav completes,
   may read wrong dept. Use `setTimeout` if calling immediately after `nav()`

3. **IPD_PATIENTS dual reference** — exists as both `const IPD_PATIENTS = []`
   and `window.IPD_PATIENTS`. Always use `window.IPD_PATIENTS` when adding
   from outside the main script closure. `renderIPD()` syncs them

4. **XREF_LOG** — session-only (not persisted to Firebase on page load, only
   written). Cross-referrals won't show in the tab after page refresh unless
   fetched from `crossReferrals/` on login

5. **printOphthoSheet field IDs** — the refraction table has two overlapping
   sets of IDs. Use these confirmed working IDs:
   - UCVA: `ucva-od-dist`, `ucva-os-dist`, `ucva-od-near`, `ucva-os-near`
   - IOP: `iop-gat-od`, `iop-gat-os`, `iop-nct-od`, `iop-nct-os`
   - Final Rx: `rf-od-sph2`, `rf-od-cyl2`, `rf-od-ax2` (and OS equivalents)
   - Cycloplegic: `cc-od-sph`, `cc-od-cyl`, `cc-od-axis`

6. **CONSENT_DATA object** — must have commas after every key-value pair.
   Previously missing comma after `ivt:` block caused the entire app to fail.
   Always validate with `node --check` after editing this section

## Development Workflow

### Making Changes
1. Open `BMH_HMS_v49.html` in a text editor
2. Make changes
3. Validate JS syntax: extract `<script>` blocks and run `node --check script.js`
4. Test in browser (Chrome/Edge recommended)
5. Increment version in `<title>` tag and internal comments

### Syntax Validation (Python)
```python
import re, subprocess, tempfile, os

with open('BMH_HMS_v49.html', 'r', encoding='utf-8') as f:
    content = f.read()

scripts = re.findall(r'<script(?:[^>]*)>(.*?)</script>', content, re.DOTALL)
for i, s in enumerate(scripts):
    if len(s) < 10:
        continue
    with tempfile.NamedTemporaryFile(mode='w', suffix='.js',
                                      delete=False, encoding='utf-8') as tf:
        tf.write(s)
        name = tf.name
    result = subprocess.run(['node', '--check', name],
                             capture_output=True, text=True)
    os.unlink(name)
    status = "OK" if result.returncode == 0 else f"ERROR: {result.stderr[:300]}"
    print(f"Script {i}: {status} ({len(s)} chars)")
```
